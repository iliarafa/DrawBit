## Reusable gotchas (carry these)

- **`PixelArtIcon` renders via SwiftUI `Canvas`, which anti-aliases the pixel blocks** — the icons
  look slightly soft and, because they reuse the same metaphors, can look like the old SF Symbols at
  a glance. They are NOT SF Symbols. (I misread this mid-session and wasted a DerivedData wipe; the
  founder corrected me.) The softness matches every other pixel icon in the app.
- **The strip's width is not frame-count-driven.** PLAY is pinned leading, ADD pinned trailing,
  frames scroll between — so "remove a frame to change the width" does nothing; the crop fix was
  viewport-snapping, not frame removal.
- **`homeScale` is load-bearing across three sites** — the open zoom, `resetView()`, and
  `isViewTransformed`. All three must use it; comparing the reset-chip gate to `1.0` breaks large
  canvases.
- **iOS `.popover` corners cannot be squared** — needs a custom overlay (the `NewPieceSheet` /
  `LayersPanel` pattern). We built one, then dropped the whole popup for tap-to-cycle.
- **This founder rejects text/mockup design proposals.** Don't pitch a redesign in words — build 2–3
  real SwiftUI variants behind a temp `-Variant N` launch arg, screenshot each on the sim, and present
  the **real captures** side by side (an **Artifact** HTML page with the PNGs inlined as base64 data
  URIs renders on their client; `show_widget` does not). And: **for a discrete control, reach for the
  app's established interaction pattern before inventing a new surface** — several rejected FPS
  redesigns were avoidable; tap-to-cycle (the existing brush idiom) was the answer all along.
- **Renaming a UI test method trips stale DerivedData** — the old method lingered in the compiled
  bundle and failed. `rm -rf ~/Library/Developer/Xcode/DerivedData/DrawBit-*` and rerun.
- **A monospaced pixel-font label in a content-sized `HStack` must be fixed character-width**, or its
  digit count jitters the whole row's distribution.
- **Screenshot workflow:** capture via an in-test `XCUIScreen.main.screenshot().pngRepresentation`
  written to the scratchpad path (temp `testCapture…` method + a `-UITest-skipLanding` launch), then
  crop with `ffmpeg -vf "crop=…,scale=…:flags=neighbor"` to keep pixels crisp. Remove the temp test
  before committing.
- **Setting a custom canvas size in a UI test** via the `NewPiece-dimField` text field is flaky (the
  delete-to-clear doesn't register; input appends and clamps to the 256 max). For the default 32 skip
  the field entirely; for the 256 extreme the append-clamp is actually reliable.
