## Release blocker (NOT code — Apple account action)
Archiving + Validate failed with two errors that are account-side:
1. **"PLA Update available"** — a new Program License Agreement is unaccepted. This locks access
   to membership resources, which blocks certificate creation.
2. **No "iOS Distribution" certificate** for team `3DLV25C9VK` — a downstream effect of #1.

**Fix:** the **Account Holder** signs in at developer.apple.com/account and accepts the updated
Program License Agreement. Then in Xcode (Settings ▸ Accounts) refresh/Download Manual Profiles
and re-run Archive ▸ Distribute; automatic signing should mint the iOS Distribution cert. If the
cert error persists, toggle "Automatically manage signing" or create an **Apple Distribution**
cert manually under Certificates. No repo change is needed.
