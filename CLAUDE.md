## Commands

The Xcode project is generated by [xcodegen](https://github.com/yonaskolb/XcodeGen) from `project.yml`. `DrawBit.xcodeproj/` is gitignored.

Prereqs: `brew install xcodegen` (Xcode 16+ already installed).

**Regenerate the project** whenever you add, rename, or move a `.swift` file — the project references source files by path and will go stale:

```bash
rm -rf DrawBit.xcodeproj && xcodegen generate
```

**Build** (simulator):

```bash
xcodebuild -project DrawBit.xcodeproj -scheme DrawBit \
  -destination 'platform=iOS Simulator,name=iPad Pro 13-inch (M5)' build
```

If the named simulator isn't installed, list available ones with `xcrun simctl list devices available | grep -i iPad` and substitute.

**Run all tests:**

```bash
xcodebuild -project DrawBit.xcodeproj -scheme DrawBit \
  -destination 'platform=iOS Simulator,name=iPad Pro 13-inch (M5)' test
```

**Run a single test class or method** (fastest feedback loop during TDD):

```bash
# whole class:
xcodebuild ... test -only-testing:DrawBitTests/FillTests
# single method:
xcodebuild ... test -only-testing:DrawBitTests/FillTests/testFillEmptyCanvasFillsAll
# UI tests (slower):
xcodebuild ... test -only-testing:DrawBitUITests
```

**Device compile-check** (no signing required):

```bash
xcodebuild -project DrawBit.xcodeproj -scheme DrawBit -configuration Release \
  -destination 'generic/platform=iOS' CODE_SIGNING_ALLOWED=NO build
```

## Architecture

The app splits into four layers with strict one-way dependencies — **UI depends on State depends on Persistence depends on Rendering/Drawing/Models**. Drawing and Rendering have no UI or SwiftData imports and are covered by fast unit tests.

```
Views (SwiftUI, UIKit input)
  └─> State (EditorState: @Observable, session-scoped, undo/redo)
        └─> Persistence (PieceRepository wraps ModelContext)
              └─> Rendering (CoreGraphics: CGImage, thumbnails, PNG)
                    └─> Drawing (pure logic: PixelGrid, tools, Bresenham)
                          └─> Models (SwiftData + value types)
```

Key boundaries to preserve when extending:

- **Pure-logic rule:** anything in `DrawBit/Drawing/` and `DrawBit/Rendering/` imports only `Foundation` / `CoreGraphics` / `ImageIO`. No SwiftUI, no SwiftData, no UIKit. This is what makes them trivially testable. New drawing tools or export paths belong here, not inside views.
- **`PieceRepository` is the only SwiftData seam.** Views call the repository; they do not touch `ModelContext` directly. If you need a new persistence operation, add a method on `PieceRepository` — don't inline SwiftData calls into a view.
- **`EditorState` is in-memory and session-scoped.** Undo/redo snapshots, current tool, current color, and view transform live here. It is cleared when the editor closes. Do not persist transform state to `Piece`.
- **Canvas transform is view-only.** `state.translation`, `state.scale`, and `state.rotation` never alter the stored pixel grid. Exported PNGs are always axis-aligned. A new feature must not "bake" transform into pixels.

## Non-obvious invariants

These are load-bearing; breaking them silently corrupts exports or kills perceived quality.

- **`PixelGrid.init(data:size:)` defensively copies its `Data` argument** (`self.data = Data(data)`). A sliced `Data` has a non-zero `startIndex`; without the copy, `data[0]` traps and `CGDataProvider` reads the wrong bytes into the CGImage. See the regression test `testSlicedDataIsNormalized`.
- **`pixelsToCGImage` uses an explicit `CGColorSpace(name: CGColorSpace.sRGB)`** — never `deviceRGB`. On iPad Pro's P3 display, `deviceRGB` would silently bake wide-gamut values into exported PNGs. The same applies to the `CGContext` created in `ThumbnailRenderer.render` and `PNGExporter.export`.
- **Nearest-neighbor everywhere:** every rendering code path sets `interpolationQuality = .none` and `setShouldAntialias(false)` on its `CGContext`, and UI `Image` views use `.interpolation(.none).antialiased(false)`. Pixel edges stay hard at any zoom.
- **Straight alpha, not premultiplied at the data level.** `PixelGrid` stores raw RGBA bytes. `CGImage` is built with `.premultipliedLast`, which is identical to straight alpha as long as alpha is only 0 or 255 — so the eraser must write `alpha = 0` exactly (`RGBA.transparent`), and drawing tools must never produce partial alpha. If that changes, the render path needs revisiting.
- **Stroke drawing uses `UIEvent.coalescedTouches` + Bresenham line interpolation.** A stroke without Bresenham leaves gaps on fast drags. New drawing tools that respond to `onStrokePoint` get this for free because `CanvasInputView.touchesMoved` already walks coalesced samples and interpolates.

## Test conventions

- Unit tests mirror the source tree: `DrawBit/Drawing/Fill.swift` → `DrawBitTests/Drawing/FillTests.swift`. Keep that mirror when adding tests.
- Unit tests `@testable import DrawBit` — internal (non-`public`) symbols are visible. Default access control is fine.
- SwiftData tests use an in-memory container: `ModelConfiguration(isStoredInMemoryOnly: true)`. See `PieceRepositoryTests` for the setup pattern; reuse it for any new persistence test.
- UI tests launch the app with `launchArguments = ["-UITest-reset"]`, which is checked in `DrawBitApp.swift` to switch the `ModelContainer` to `inMemory: true`. Tests get a clean store every run.
